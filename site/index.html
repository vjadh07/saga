The transaction layer for AI agents.

Saga makes every action an agent takes crash-safe, verified against the vendor, and undoable, on an append-only ledger.

Agents got hands. Nobody gave them a spine.

Hundreds of agent products can now book, pay, send, and cancel. Three failures follow them everywhere.

They die mid-action

A crash between the charge and the confirmation leaves a half-finished job, and the restarted agent has no memory of what it already did. Retry double-books. Caution loses the booking.

Vendors lie

The API does the work, then answers 500. The agent believes the error and pays twice.

Nothing is provable

When finance asks what the agent did last month, a chat log is not an answer.

How it works

Databases solved this fifty years ago with write-ahead logs. Saga applies the same discipline to a new unreliable node: an LLM with a credit card.

STAGE

Intent hits the ledger before any network call leaves the machine. If the process dies now, the plan survives it.

→
VERIFY

The vendor's response is never trusted. Saga asks the vendor's own records whether the effect exists.

→
COMMIT

Only confirmed effects commit. Everything else retries once, aborts durably, or compensates in reverse order.

STAGED → CALLED → RECONCILED → COMMITTED   | cancel: COMPENSATION_CALLED → COMPENSATED   | dead end: ABORTED

Kill it. It finishes anyway.

A real SIGKILL lands mid-booking. On restart, the engine reconciles against the vendor and completes exactly once, before the model says a word.

$ CRASH_AFTER="hotel.book:CALLED" npm run agent -- "book my trip"
-> book_flight {"flightId":"F100"} COMMITTED
-> book_hotel {"hotelId":"H100"} killed: exit 137

$ npm run agent -- "finish booking my trip"
[recovery] reconciled 1 in-flight action left by a previous life
trip: committed. flight, hotel, calendar: exactly one of each.

The ledger audits itself.

A second agent on the same substrate reads the ledger against vendor records and surfaces every reconciliation break, with evidence.

$ npm run audit -- "investigate the hotels vendor"
-> run_reconciliation {"vendor":"hotels"}
5 reconciliation breaks. Vendor holds 2 bookings the ledger never authorized,
1 duplicate charge for an authorized stay, and still holds a booking
the ledger says was compensated six days earlier. Report saved.

Use it

Clone it, kill it, audit it. Node 22.5 or newer and a logged-in Claude Code install are the only requirements.

Run the whole thing

git clone https://github.com/vjadh07/saga
cd saga && npm install
npm test          # 60+ tests, incl. a real kill -9 round trip
npm run vendors   # mock vendors on :4100
npm run viewer    # live ledger on :4200

npm run agent -- "book me a flight to SFO next friday"
CRASH_AFTER="hotel.book:CALLED" npm run agent -- "book my trip"
npm run agent -- "finish booking my trip"

npm run seed      # a month of history, 5 planted breaks
npm run audit -- "investigate the hotels vendor"

Put your own tool on the ledger

// any side effect becomes a saga action:
// implement call / reconcile / compensate for your vendor
const staged = saga.stage({
  sagaId: "job-42",
  type: "invoice.send",
  vendor: "billing",
  params: { invoiceId: "inv_881" },
});

await saga.execute(staged.actionId);
// STAGED -> CALLED -> RECONCILED -> COMMITTED,
// or it does not commit at all

await saga.cancel("job-42");   // verified undo, newest first
saga.receipt("job-42");        // the whole story, forever
